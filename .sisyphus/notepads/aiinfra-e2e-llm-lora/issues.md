- 2026-04-13 ManagedVLLMServer subprocess IO review: `src/aiinfra_e2e/serve/vllm_server.py` launches vLLM via `subprocess.Popen(..., stdout=subprocess.PIPE, stderr=subprocess.PIPE, text=True)` in `ManagedVLLMServer.start` (lines 125-131) but has no background drain/consumer for either stream while the process runs. Only `stderr.read()` is called in `wait_until_ready` after `poll()` indicates process exit (lines 138-142). This can deadlock startup: if vLLM writes enough logs to fill OS pipe buffer before readiness, child blocks on write, server never reaches ready state, and wrapper eventually times out.
- Minimal no-dependency fixes: (A) safest/smallest: remove PIPEs and inherit parent streams (`stdout=None, stderr=None`), (B) redirect both to append log file handle(s) opened by wrapper, or (C) keep PIPEs but start 2 daemon drain threads (stdout/stderr) that continuously read lines until EOF.
